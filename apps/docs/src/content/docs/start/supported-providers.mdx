---
title: Supported providers
description: Current, upcoming, and planned platform targets for kubwave installs.
---

kubwave installs onto Kubernetes clusters. Provider support means the CLI knows the
right platform defaults for load balancers, storage, node selectors, and install-time
checks.

Provider selection is cluster-wide: one kubwave cluster uses exactly one provider
profile. To run kubwave on another provider, use a separate Kubernetes cluster.

## Supported today

| Provider             | Platform ID          | Status    | Notes                                             |
| -------------------- | -------------------- | --------- | ------------------------------------------------- |
| Cloudfleet (Hetzner) | `cloudfleet-hetzner` | Supported | First-class target for production installs today. |

Use this target with:

```sh
kubwave install --platform cloudfleet-hetzner
```
